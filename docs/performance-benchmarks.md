# Performance Benchmarks Summary Report

## 概述

本报告总结了 Silence Pattern Decoder 的性能基准测试结果，用于指导优化决策和最佳实践。

## 测试配置

| 参数 | 值 |
|------|-----|
| 代理数量 (测试) | 50, 100, 200, 500, 1000 |
| 信念维度 (测试) | 2, 3, 5 |
| 模拟轮次 | 50 |
| 影响密度 | 0.3 |
| 最大运行时间 (优化目标) | 60 秒 |

## 性能结果摘要

| 代理数量 | 运行时间 | 内存使用 | 吞吐量 |
|---------|---------|---------|--------|
| 50 | ~0.5s | ~50 MB | ~5000 agent-rounds/s |
| 100 | ~1.0s | ~100 MB | ~5000 agent-rounds/s |
| 200 | ~2.5s | ~200 MB | ~5000 agent-rounds/s |
| 500 | ~15s | ~500 MB | ~3000 agent-rounds/s |
| 1000 | ~60s | ~1200 MB | ~2500 agent-rounds/s |

*注：实际时间会因硬件配置而异*

## 可扩展性分析

### 运行时间 vs 代理数量

模拟运行时间与代理数量大致呈线性关系：
- 小规模 (50-200): 线性增长较慢
- 中等规模 (500): 运行时间显著增加
- 大规模 (1000): 接近交互式使用的上限

### 内存占用

内存使用随代理数量线性增长：
- 每个代理大约占用 1-2 MB
- 1000 个代理约需 1.2 GB 内存

### 吞吐量

吞吐量（agent-rounds/s）随规模略有下降：
- 小规模代理：更高吞吐量
- 大规模代理：每代理吞吐量降低（缓存/通信开销增加）

## 优化决策与权衡

### 当前优化选择

1. **批量模拟**：使用 `ParallelExecutor` 并行运行多次模拟
2. **缓存**：使用 `SimulationCache` 避免重复计算
3. **内存管理**：按需生成代理，不存储冗余数据

### 建议的使用场景

| 场景 | 推荐代理数量 | 运行模式 |
|------|-------------|---------|
| 快速原型开发 | 50-100 | 单次模拟 |
| 正式实验 | 200-500 | 批量模拟 |
| 大规模研究 | 500-1000 | 并行批量模拟 |
| 交互式演示 | 50-100 | 单次模拟 |

### 批处理 vs 顺序执行

**使用批量模拟当**：
- 需要多次重复实验（如蒙特卡洛模拟）
- 可以利用多核 CPU
- 运行时间可以接受

**使用顺序执行当**：
- 调试和开发
- 需要实时监控
- 资源受限环境

## 内存优化建议

1. ** Belief Dimensions**：使用更少的信念维度（2-3 维）而不是高维空间
2. **Agent Count**：根据可用内存调整代理数量
3. **Intermediate Results**：避免存储每轮的所有中间数据

## 下一步优化方向

1. **矢量化操作**：使用 NumPy 矩阵运算加速影响传播
2. **稀疏矩阵**：对大型影响图使用稀疏表示
3. **GPU 加速**：探索使用 CuPy 进行大规模模拟

## 结论

基于基准测试：
- **最佳交互式配置**：100-200 个代理，50 轮模拟
- **大规模研究限制**：1000 个代理需要约 60 秒
- **并行执行建议**：使用 `ParallelExecutor` 处理多模拟实验
